Add ``FanOutMapper`` for one-to-many partition mapping (e.g. one weekly upstream key to seven daily downstream Dag runs). It composes ``upstream_mapper + window + fine_mapper``, mirroring the shape of ``RollupMapper`` and reusing the existing ``Window`` classes (``DayWindow``, ``WeekWindow``, ``MonthWindow``, ``QuarterWindow``, ``YearWindow``). A new ``[scheduler] partition_fanout_max_keys`` config caps the number of downstream keys produced per upstream event.
